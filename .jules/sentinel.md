## 2026-04-08 - Fixed XSS vulnerability in marketplace.html
**Vulnerability:** User-controlled extension metadata (like names, descriptions, and publishers) from the Open VSX registry was being directly interpolated into the `.innerHTML` string without any escaping in `views/marketplace.html`. This created a critical Cross-Site Scripting (XSS) vulnerability.
**Learning:** In a vanilla HTML/JS application, assigning raw template strings to `.innerHTML` is a significant risk. The vulnerability was compounded because dynamic variables were also being placed inside inline event handlers (`onclick` and `onerror`), where simple HTML escaping is insufficient because the browser decodes HTML entities *before* executing the JavaScript context.
**Prevention:** Always use a helper function to escape dynamic text before it reaches `.innerHTML`. Crucially, avoid inline JavaScript event handlers (like `onclick="..."`). Instead, attach event listeners directly in JS using `element.addEventListener` or `element.onclick` and pass necessary state via safe `data-*` attributes.
