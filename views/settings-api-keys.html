Guardrails

Keeping automation safe

Credentials are only as safe as the habits around them.

  • Keep production route-write access separate from read-only metrics keys so observation does not imply mutation.
  • Assign one key per automation workflow so revocation does not force unrelated systems to rotate at the same time.
  • Prefer region-specific tokens during staged rollouts to limit the blast radius of misrouted deploy jobs.
  • After a manual hotfix or operator override, rotate the key involved and record why the extra privilege was needed.
