# Requirements: Mesh v2.2 — Live App Bug Fix & Editor Overhaul (Continued)

**Defined:** 2026-04-19
**Core Value:** Fix the remaining broken surfaces in the Mesh IDE that were deferred from v2.1 — marketplace search, settings auth, voice synthesis, UI polish, and .mesh content quality.

## v1 Requirements

### Terminal

- [ ] **TERM-04**: Server terminal works even when node-pty is unavailable — fallback shell (child_process) spawns with visible warning instead of dead connection
- [ ] **TERM-05**: Local agent proxy check runs before PTY availability check — users with a local agent get a terminal regardless of server-side PTY state

### Marketplace

- [ ] **MKT-01**: Extension search queries are proxied through the server — browser does not call Open VSX API directly (avoids CSP/CORS issues in production)
- [ ] **MKT-02**: Extension display shows name, publisher, version, description, and install button — consistent with VS Code marketplace UX

### Settings

- [ ] **SETT-04**: Settings page loads correctly when the user is already authenticated — no spurious redirect to login when navigating from workspace
- [ ] **SETT-05**: App theme defaults to "system" (follows OS dark/light preference) on first load when no preference has been saved

### Voice Agent

- [ ] **VOIC-03**: Voice agent TTS output uses Amazon Polly neural voices — Azure TTS dependency fully removed, Polly integration complete end-to-end

### UI Elements

- [ ] **UIEL-07**: Flash of unstyled content (FOUC) eliminated — correct theme applied before first paint on all pages
- [ ] **UIEL-08**: "Indexing..." status indicator only shows when workspace indexing is actively running — not on fresh load or when no folder is open

### .mesh Folder

- [ ] **MESH-02**: `.mesh/files.md` includes per-file role descriptions derived from the file roles table built in Phase 45 (CAP-04)
- [ ] **MESH-03**: `.mesh/project.json` includes the symbol count and top exported symbols for each file
- [ ] **MESH-04**: `.mesh/rules.md` content is meaningful and workspace-specific — not a generic template
