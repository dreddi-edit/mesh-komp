# Plan 26-02: Nunjucks Templates + esbuild Pipeline

<objective>
Convert 16 standalone HTML pages to nunjucks template inheritance with shared base layout. Bundle frontend assets with esbuild. Vendor animejs into bundle.
</objective>

<tasks>

<task id="1" name="Set up nunjucks and create base layout">
<read_first>
- package.json
- src/server.js
- views/app.html
</read_first>
<action>
1. Run `npm install nunjucks`
2. Configure nunjucks in `src/server.js`:
   ```javascript
   const nunjucks = require('nunjucks');
   nunjucks.configure('views', {
     autoescape: true,
     express: app,
     watch: config.NODE_ENV === 'development',
   });
   app.set('view engine', 'njk');
   ```

3. Create `views/layouts/base.njk` — shared base template:
   ```html
   <!DOCTYPE html>
   <html lang="en">
   <head>
     <meta charset="UTF-8">
     <meta name="viewport" content="width=device-width, initial-scale=1.0">
     <title>{% block title %}Mesh.{% endblock %}</title>
     <link rel="stylesheet" href="/assets/tokens.css">
     {% block styles %}{% endblock %}
     {% block head %}{% endblock %}
   </head>
   <body>
     {% block nav %}{% endblock %}
     {% block content %}{% endblock %}
     {% block scripts %}{% endblock %}
   </body>
   </html>
   ```

4. Update `sendHtmlWithHashes` to use `res.render()` for .njk templates, passing nonce and asset hashes as template variables.
</action>
<acceptance_criteria>
- `grep "nunjucks" package.json` matches
- File `views/layouts/base.njk` exists
- `grep "nunjucks.configure" src/server.js` matches
</acceptance_criteria>
</task>

<task id="2" name="Convert HTML pages to nunjucks templates">
<read_first>
- views/app.html
- views/index.html
- views/terminal.html
</read_first>
<action>
Convert all 16 HTML pages from standalone `.html` to `.njk` extending base layout:

For each page:
1. Rename `.html` to `.njk`
2. Replace full HTML boilerplate with `{% extends "layouts/base.njk" %}`
3. Move page-specific content into `{% block content %}...{% endblock %}`
4. Move page-specific styles into `{% block styles %}...{% endblock %}`
5. Move page-specific scripts into `{% block scripts %}...{% endblock %}`
6. Add nonce to inline scripts/styles: `nonce="{{ cspNonce }}"`

Update route view references from `.html` to `.njk` in route files and server.js.
</action>
<acceptance_criteria>
- All 16 views are `.njk` files extending base layout
- `grep "extends" views/app.njk` matches
- Route files reference `.njk` templates
- `npm test` passes
</acceptance_criteria>
</task>

<task id="3" name="Set up esbuild and vendor animejs">
<read_first>
- package.json
- assets/
</read_first>
<action>
1. Run `npm install --save-dev esbuild`
2. Create `esbuild.config.js`:
   ```javascript
   const esbuild = require('esbuild');
   esbuild.build({
     entryPoints: ['assets/app.js', 'assets/app-workspace.js', 'assets/app-graph.js', 'assets/settings.js'],
     bundle: true,
     outdir: 'dist/assets',
     minify: process.env.NODE_ENV === 'production',
     sourcemap: true,
     format: 'iife',
     target: ['es2020'],
     loader: { '.css': 'css' },
     entryNames: '[name]-[hash]',
   });
   ```
3. Add scripts to package.json:
   ```json
   "build": "node esbuild.config.js",
   "build:watch": "node esbuild.config.js --watch"
   ```
4. Vendor animejs: import animejs in relevant JS files instead of loading from node_modules via Express static
5. Update server.js to serve from `dist/assets/` in production, `assets/` in development
6. Update asset hash map to work with esbuild output filenames (they include content hashes)
</action>
<acceptance_criteria>
- `grep "esbuild" package.json` matches
- File `esbuild.config.js` exists
- `npm run build` produces files in `dist/assets/`
- `grep "animejs" package.json` in dependencies (already there), no longer served from node_modules
- `npm test` passes
</acceptance_criteria>
</task>
