# Plan 26-01: CSS Design Tokens + Responsive Foundation

<objective>
Extract CSS design tokens as custom properties on :root, replace hardcoded values across all stylesheets, and add responsive breakpoints for 768px minimum.
</objective>

<tasks>

<task id="1" name="Create design token CSS file">
<read_first>
- assets/app-workspace.css
- assets/mesh-settings.css
</read_first>
<action>
Create `assets/tokens.css` with `:root` custom properties:

```css
:root {
  /* Colors — extracted from existing stylesheets */
  --color-bg-primary: #1e1e2e;
  --color-bg-secondary: #181825;
  --color-bg-surface: #313244;
  --color-bg-elevated: #45475a;
  --color-text-primary: #cdd6f4;
  --color-text-secondary: #a6adc8;
  --color-text-muted: #6c7086;
  --color-accent-primary: #89b4fa;
  --color-accent-secondary: #74c7ec;
  --color-accent-success: #a6e3a1;
  --color-accent-warning: #f9e2af;
  --color-accent-error: #f38ba8;
  --color-border: #585b70;

  /* Spacing */
  --space-xs: 0.25rem;
  --space-sm: 0.5rem;
  --space-md: 1rem;
  --space-lg: 1.5rem;
  --space-xl: 2rem;
  --space-2xl: 3rem;

  /* Typography */
  --font-sans: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
  --font-mono: 'JetBrains Mono', 'Fira Code', 'Cascadia Code', monospace;
  --font-size-xs: 0.75rem;
  --font-size-sm: 0.875rem;
  --font-size-md: 1rem;
  --font-size-lg: 1.125rem;
  --font-size-xl: 1.25rem;
  --font-size-2xl: 1.5rem;
  --line-height-tight: 1.25;
  --line-height-normal: 1.5;

  /* Shadows */
  --shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.3);
  --shadow-md: 0 4px 6px rgba(0, 0, 0, 0.4);
  --shadow-lg: 0 10px 15px rgba(0, 0, 0, 0.5);

  /* Border radius */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
  --radius-full: 9999px;

  /* Breakpoints (as reference; used in media queries) */
  --breakpoint-sm: 640px;
  --breakpoint-md: 768px;
  --breakpoint-lg: 1024px;
  --breakpoint-xl: 1280px;

  /* Z-index layers */
  --z-dropdown: 100;
  --z-modal: 200;
  --z-toast: 300;
  --z-tooltip: 400;
}
```

Extract actual color values from existing CSS files — scan for hex codes, rgb values, and map to semantic token names.
</action>
<acceptance_criteria>
- File `assets/tokens.css` exists
- `grep ":root" assets/tokens.css` matches
- `grep "color-bg-primary" assets/tokens.css` matches
- `grep "font-mono" assets/tokens.css` matches
- `grep "space-md" assets/tokens.css` matches
</acceptance_criteria>
</task>

<task id="2" name="Replace hardcoded values in stylesheets">
<read_first>
- assets/app-workspace.css
- assets/mesh-docs.css
- assets/mesh-settings.css
- assets/repo-docs.css
</read_first>
<action>
For each CSS file:

1. Replace hardcoded color values (`#1e1e2e`, `#cdd6f4`, etc.) with `var(--color-*)` tokens
2. Replace hardcoded spacing (`8px`, `16px`, `24px`) with `var(--space-*)` tokens
3. Replace hardcoded font families with `var(--font-*)` tokens
4. Replace hardcoded font sizes with `var(--font-size-*)` tokens
5. Replace hardcoded border-radius with `var(--radius-*)` tokens
6. Replace hardcoded shadows with `var(--shadow-*)` tokens

Include `@import './tokens.css';` or add `<link>` tag in HTML pages for tokens.css.
</action>
<acceptance_criteria>
- `grep -c "var(--" assets/app-workspace.css` returns significant count
- Minimal hardcoded hex color values remaining in stylesheets
- Pages render with same visual appearance (tokens map to original values)
</acceptance_criteria>
</task>

<task id="3" name="Add responsive media queries">
<read_first>
- assets/app-workspace.css
</read_first>
<action>
Add responsive breakpoints to key layout elements:

1. At bottom of each CSS file, add media queries for 768px minimum width
2. Focus on:
   - Sidebar collapse at small screens
   - Panel stacking (side-by-side → stacked)
   - Font size adjustments
   - Spacing reductions
3. Pattern:
   ```css
   @media (max-width: 768px) {
     .sidebar { display: none; }
     .main-content { width: 100%; }
     .panel-group { flex-direction: column; }
   }
   ```
</action>
<acceptance_criteria>
- `grep "@media" assets/app-workspace.css` matches
- `grep "768px" assets/app-workspace.css` matches
- Pages usable at 768px width
</acceptance_criteria>
</task>

</tasks>
