# Plan 26-03: Accessibility + Visual Verification

<objective>
Add ARIA roles, keyboard navigation, and visible focus indicators to custom UI chrome. Verify all 16 pages work correctly with templates and build pipeline.
</objective>

<tasks>

<task id="1" name="Add ARIA roles to custom UI chrome">
<read_first>
- assets/app-workspace.js
- assets/app.js
- assets/settings.js
</read_first>
<action>
Add ARIA attributes to custom UI components:

1. **Tabs:** `role="tablist"` on container, `role="tab"` on each tab, `role="tabpanel"` on panels, `aria-selected="true/false"` on active/inactive tabs
2. **Panels:** `role="region"` with `aria-label` describing content
3. **Modals:** `role="dialog"` with `aria-modal="true"`, `aria-labelledby` pointing to title
4. **Context menus:** `role="menu"`, `role="menuitem"` on items
5. **Tree views (file tree):** `role="tree"`, `role="treeitem"`, `aria-expanded` for folders
6. **Status bar:** `role="status"` with `aria-live="polite"`

Add keyboard navigation:
- Tabs: Arrow keys to move between tabs, Enter/Space to activate
- Modals: Escape to close, Tab trapped within modal
- Context menus: Arrow keys to navigate, Enter to select, Escape to close
- Tree views: Arrow keys to navigate, Enter to open/close
</action>
<acceptance_criteria>
- `grep 'role="tablist"' assets/app-workspace.js` matches
- `grep 'role="dialog"' assets/app.js` matches
- `grep "aria-" assets/app-workspace.js` returns multiple matches
- Keyboard navigation works for tabs, modals, context menus
</acceptance_criteria>
</task>

<task id="2" name="Add visible focus indicators">
<read_first>
- assets/tokens.css
</read_first>
<action>
Add focus styles to `assets/tokens.css`:

```css
/* Focus indicators */
:focus-visible {
  outline: 2px solid var(--color-accent-primary);
  outline-offset: 2px;
}

button:focus-visible,
[role="tab"]:focus-visible,
[role="menuitem"]:focus-visible,
[role="treeitem"]:focus-visible,
a:focus-visible {
  outline: 2px solid var(--color-accent-primary);
  outline-offset: 2px;
  border-radius: var(--radius-sm);
}

/* Remove default outline when :focus-visible is supported */
:focus:not(:focus-visible) {
  outline: none;
}
```
</action>
<acceptance_criteria>
- `grep "focus-visible" assets/tokens.css` matches
- Focus indicators visible when tabbing through UI
</acceptance_criteria>
</task>
