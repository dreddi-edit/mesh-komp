# Plan 27-02: CloudWatch Error Monitoring

<objective>
Add CloudWatch metric filters for typed error codes from Phase 19's error class hierarchy. Build on Phase 12's existing CloudWatch setup.
</objective>

<tasks>

<task id="1" name="Add error-code metric filters to CloudFormation">
<read_first>
- infra/cloudformation.yml
- src/errors/index.js
- src/logger.js
</read_first>
<action>
In `infra/cloudformation.yml`, add CloudWatch MetricFilter resources for each error code:

1. Find the existing Log Group resource (from Phase 12): `/mesh/app`
2. Add MetricFilters that parse the JSON log `code` field:

```yaml
  ValidationErrorMetricFilter:
    Type: AWS::Logs::MetricFilter
    Properties:
      LogGroupName: /mesh/app
      FilterPattern: '{ $.code = "VALIDATION_ERROR" }'
      MetricTransformations:
        - MetricName: ValidationErrors
          MetricNamespace: Mesh/Errors
          MetricValue: "1"
          DefaultValue: 0

  NotFoundErrorMetricFilter:
    Type: AWS::Logs::MetricFilter
    Properties:
      LogGroupName: /mesh/app
      FilterPattern: '{ $.code = "NOT_FOUND" }'
      MetricTransformations:
        - MetricName: NotFoundErrors
          MetricNamespace: Mesh/Errors
          MetricValue: "1"
          DefaultValue: 0

  AuthErrorMetricFilter:
    Type: AWS::Logs::MetricFilter
    Properties:
      LogGroupName: /mesh/app
      FilterPattern: '{ $.code = "AUTH_ERROR" }'
      MetricTransformations:
        - MetricName: AuthErrors
          MetricNamespace: Mesh/Errors
          MetricValue: "1"
          DefaultValue: 0

  InternalErrorMetricFilter:
    Type: AWS::Logs::MetricFilter
    Properties:
      LogGroupName: /mesh/app
      FilterPattern: '{ $.code = "INTERNAL_ERROR" }'
      MetricTransformations:
        - MetricName: InternalErrors
          MetricNamespace: Mesh/Errors
          MetricValue: "1"
          DefaultValue: 0

  ConflictErrorMetricFilter:
    Type: AWS::Logs::MetricFilter
    Properties:
      LogGroupName: /mesh/app
      FilterPattern: '{ $.code = "CONFLICT" }'
      MetricTransformations:
        - MetricName: ConflictErrors
          MetricNamespace: Mesh/Errors
          MetricValue: "1"
          DefaultValue: 0
```

3. Add a CloudWatch Alarm for 5xx error rate:
```yaml
  HighErrorRateAlarm:
    Type: AWS::CloudWatch::Alarm
    Properties:
      AlarmName: mesh-high-error-rate
      MetricName: InternalErrors
      Namespace: Mesh/Errors
      Statistic: Sum
      Period: 300
      EvaluationPeriods: 2
      Threshold: 10
      ComparisonOperator: GreaterThanThreshold
      TreatMissingData: notBreaching
```
</action>
<acceptance_criteria>
- `grep "ValidationErrorMetricFilter" infra/cloudformation.yml` matches
- `grep "AuthErrorMetricFilter" infra/cloudformation.yml` matches
- `grep "InternalErrorMetricFilter" infra/cloudformation.yml` matches
- `grep "Mesh/Errors" infra/cloudformation.yml` matches
- `grep "HighErrorRateAlarm" infra/cloudformation.yml` matches
</acceptance_criteria>
</task>

</tasks>
