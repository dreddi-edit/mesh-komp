# Plan 41-01: Add Inline Flash-Prevention Script to app.njk

**Objective:** Eliminate the flash of wrong theme on `/app` page load by inserting an inline `<script>` as the very first element of `{% block head %}` in `views/app.njk`. The script reads `localStorage.meshAppearance` (primary) and `localStorage.meshSettings` (fallback), resolves `'system'` via `matchMedia`, and sets `document.documentElement.dataset.theme` synchronously before any CSS paints.

**Wave:** 1 (autonomous — single template insertion, no dependencies)

---

## Task 41-01-01: Insert inline theme flash-prevention script in app.njk

<task id="41-01-01">
<title>Add inline theme script as first element of {% block head %} in app.njk</title>

<read_first>
- views/app.njk (read lines 1–30 — confirm {% block head %} starts at line 5, and the current first child is a <style> block)
- views/settings.njk (line 23 — reference implementation of the correct inline script; uses meshAppearance key with system fallback)
- views/layouts/base.njk (confirm <head> is in base.njk — the {% block head %} block is injected into it, so scripts added to the block run inside <head>)
</read_first>

<action>
In `views/app.njk`, insert the following inline script as the **first line** inside `{% block head %}`, immediately before the `<style>` block for `#page-transition-overlay` (currently the first element at line 6).

Insert this exact script on a new line after `{% block head %}` (line 5):

```html
<script>(function(){try{var a=JSON.parse(localStorage.getItem('meshAppearance')||'{}');var m=JSON.parse(localStorage.getItem('meshSettings')||'{}');var t=String(a.theme||m.theme||'system').trim();if(t==='system')t=window.matchMedia&&window.matchMedia('(prefers-color-scheme:dark)').matches?'dark':'light';document.documentElement.dataset.theme=t==='dark'?'dark':'light';}catch(e){document.documentElement.dataset.theme='light';}})();</script>
```

The result should look like:
```
{% block head %}
<script>(function(){...})();</script>
<style>
  #page-transition-overlay {
```

Do NOT move, modify, or remove the existing `<style>` block or the pathname-normalization `<script>` that follows. Only insert the new line.
</action>

<acceptance_criteria>

- `grep -n "meshSettings" views/app.njk` → at least 1 match (fallback read)
- `grep -n "prefers-color-scheme" views/app.njk` → at least 1 match
- `grep -n "||'system'" views/app.njk` → at least 1 match
- The script line appears BEFORE the `#page-transition-overlay` `<style>` block (verify line ordering with `grep -n`)
- `grep -n "meshAppearance\|#page-transition-overlay" views/app.njk` — meshAppearance line number < page-transition-overlay line number
- The existing pathname-normalization script (`p.endsWith('.html')`) is still present and unmodified
</acceptance_criteria>
