# Plan 32-02: Listen Loop Fix

## Goal

After a voice response, the session immediately re-enters `listening` state. Ambient noise above the 0.012 RMS threshold triggers VAD → empty transcription → `streamSpeechResponse('I did not catch that...')` → `response.done` → repeat. 

Fix: (1) backend dead zone prevents audio processing for 1500ms after response delivery; (2) frontend transitions to new `ready` state instead of `listening` after `response.done`; (3) empty transcription sends a visual-only event instead of spoken feedback.

## Files to Read Before Implementing

- `src/routes/realtime.routes.js` lines 327–497
- `assets/features/voice-chat.js` lines 206–214, 459–527, 692–700
