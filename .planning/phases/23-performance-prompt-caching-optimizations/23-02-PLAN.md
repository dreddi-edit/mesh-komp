# Plan 23-02: Async HTML Serving + Parallel Enrichment

<objective>
Replace sync file reads on the request path with async reads + in-memory cache. Add bounded concurrency to workspace enrichment for parallel file processing.
</objective>

<task id="1" name="Async HTML serving with cache">
<read_first>
- src/server.js
</read_first>
<action>
In `src/server.js`:

1. Find `sendHtmlWithHashes()` function (approx line 177) which uses `fs.readFileSync`
2. Create a module-level cache: `const htmlCache = new Map();`
3. Convert to async:
   ```javascript
   async function sendHtmlWithHashes(res, viewPath, assetHashes, nonce) {
     let html = htmlCache.get(viewPath);
     if (!html) {
       html = await fs.promises.readFile(viewPath, 'utf8');
       // Apply asset hash replacements
       for (const [name, hash] of Object.entries(assetHashes)) {
         html = html.replace(new RegExp(name, 'g'), hash);
       }
       htmlCache.set(viewPath, html);
     }
     // Apply per-request nonce if present
     let output = html;
     if (nonce) {
       output = html.replace(/<script/g, `<script nonce="${nonce}"`);
       output = output.replace(/<style/g, `<style nonce="${nonce}"`);
     }
     res.type('html').send(output);
   }
   ```
4. Startup-time sync reads (buildAssetHashMap, buildViewRouteMap) are fine — only fix request-path reads
5. Cache invalidation: in development mode, skip cache (always read from disk). In production, cache is permanent (server restart refreshes).
</action>

In `src/core/workspace-ops.js`, find `enrichLocalWorkspaceRecords()` (approx lines 40-51):

1. Current code processes files serially within a batch
2. Add bounded concurrency using a simple batch limiter:
   ```javascript
   const ENRICHMENT_CONCURRENCY = config.ENRICHMENT_CONCURRENCY || 8;

   async function enrichLocalWorkspaceRecords(records) {
     const results = [];
     for (let i = 0; i < records.length; i += ENRICHMENT_CONCURRENCY) {
       const batch = records.slice(i, i + ENRICHMENT_CONCURRENCY);
       const batchResults = await Promise.all(
         batch.map(record => enrichSingleRecord(record))
       );
       results.push(...batchResults);
     }
     return results;
   }
   ```
3. Add `ENRICHMENT_CONCURRENCY` to `src/config/index.js`:
   ```javascript
   ENRICHMENT_CONCURRENCY: parseIntegerInRange(env.ENRICHMENT_CONCURRENCY, 1, 32) || 8,
   ```
</action>
