# Plan 23-01: Anthropic + Bedrock Prompt Caching

<objective>
Add prompt caching to Anthropic native and Bedrock streaming, convert Bedrock client to module-level singleton, and fix maxTokens to respect user preference.
</objective>

<tasks>

<task id="1" name="Add Anthropic prompt caching headers">
<read_first>
- src/core/model-providers.js
</read_first>
<action>
In `src/core/model-providers.js`, find the `streamAnthropicNative` function:

1. Add beta header to request: `'anthropic-beta': 'prompt-caching-2024-07-31'`
2. Add `cache_control` block to system messages:
   ```javascript
   system: [{
     type: 'text',
     text: systemPrompt,
     cache_control: { type: 'ephemeral' }
   }]
   ```
3. This tells Anthropic's API to cache the system prompt across requests within the same conversation
</action>
<acceptance_criteria>
- `grep "prompt-caching" src/core/model-providers.js` matches
- `grep "cache_control" src/core/model-providers.js` matches
- `grep "ephemeral" src/core/model-providers.js` matches
</acceptance_criteria>
</task>

<task id="2" name="Add Bedrock prompt caching and singleton">
<read_first>
- src/core/model-providers.js
</read_first>
<action>
In `src/core/model-providers.js`:

1. Find where `BedrockRuntimeClient` is instantiated (likely per-request)
2. Move to module level: `let bedrockClient = null;` then lazy-init:
   ```javascript
   function getBedrockClient() {
     if (!bedrockClient) {
       bedrockClient = new BedrockRuntimeClient({ region: config.AWS_REGION });
     }
     return bedrockClient;
   }
   ```
3. Replace all per-request instantiations with `getBedrockClient()`
4. Add `cache_control` array on system block in Bedrock streaming payload:
   ```javascript
   system: [{ text: systemPrompt, cache_control: [{ type: 'default' }] }]
   ```
</action>
<acceptance_criteria>
- `grep "getBedrockClient" src/core/model-providers.js` matches
- `grep "let bedrockClient" src/core/model-providers.js` matches
- BedrockRuntimeClient instantiated at most once per process
- `npm test` passes
</acceptance_criteria>
</task>

<task id="3" name="Fix maxTokens to respect user preference">
<read_first>
- src/core/model-providers.js
</read_first>
<action>
In `streamAnthropicNative`:

1. Find where `max_tokens` is set in the API request payload
2. Change to: `max_tokens: storedCredentials?.anthropic?.maxTokens || 4096`
3. The credential flow is: DynamoDB -> getStoredCredentialsForUser() -> credential cache -> mergeChatCredentials() -> provider call

</action>
<acceptance_criteria>

</task>

</tasks>
