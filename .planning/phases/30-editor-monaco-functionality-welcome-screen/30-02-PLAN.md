# Plan 30-02: Fix Monaco Loader Race Condition

## Objective

Replace the brittle one-shot `typeof require === 'undefined'` guard in `initMonaco()` with a polling loop that waits up to 8 seconds for the Monaco AMD loader to become available. This eliminates the silent failure that causes inconsistent editor rendering (blank editor or no syntax highlighting).

## Context

`initMonaco()` at `assets/app-workspace.js:1132` currently reads:
```js
function initMonaco(cb){if(typeof require==='undefined')return;require.config({paths:{vs:'https://cdn.jsdelivr.net/npm/monaco-editor@0.52.2/min/vs'}});require(['vs/editor/editor.main'],()=>{S.monacoReady=true;cb();});}
```

The `loader.js` CDN script (app.njk:521) sets `window.require` (AMD loader). Both `loader.js` and `app-workspace.js` use `defer`, so they fire in DOM order after parse. The race occurs on slow/unreliable CDN connections where `loader.js` hasn't finished executing when `initMonaco` runs. The hard early return means Monaco never initializes for that page load.

The Monaco worker config (`MonacoEnvironment.getWorkerUrl` at `app.njk:507-519`) is correct and must not be changed.

<action>
In `assets/app-workspace.js`, replace the single-line `initMonaco` function at line 1132 with the following multi-line implementation.

**Find this exact line:**
```js
function initMonaco(cb){if(typeof require==='undefined')return;require.config({paths:{vs:'https://cdn.jsdelivr.net/npm/monaco-editor@0.52.2/min/vs'}});require(['vs/editor/editor.main'],()=>{S.monacoReady=true;cb();});}
```

**Replace with:**
```js
function initMonaco(cb) {
  const VS_BASE = 'https://cdn.jsdelivr.net/npm/monaco-editor@0.52.2/min/vs';
  const POLL_INTERVAL_MS = 50;
  const MAX_WAIT_MS = 8000;
  let elapsed = 0;
  const tid = setInterval(() => {
    elapsed += POLL_INTERVAL_MS;
    if (typeof require !== 'undefined') {
      clearInterval(tid);
      require.config({ paths: { vs: VS_BASE } });
      require(['vs/editor/editor.main'], () => { S.monacoReady = true; cb(); });
    } else if (elapsed >= MAX_WAIT_MS) {
      clearInterval(tid);
    }
  }, POLL_INTERVAL_MS);
}
```

No changes to `createEditor()`, `registerInlineCompletions()`, or any other function. The `cb` parameter contract is unchanged — callers pass the same callback as before.
</action>
