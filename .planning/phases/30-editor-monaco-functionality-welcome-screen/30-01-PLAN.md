# Plan 30-01: Fix False Indexing Indicator

## Objective

Prevent the `#idxProgWrap` status bar indicator from appearing when no folder is open. Add a single guard to `updateIndexProgressState` that blocks all non-idle state changes when `S.dirHandle` is not set.

## Context

`updateIndexProgressState()` at `assets/app-workspace.js:106` controls the indexing progress bar in the status bar. The function is called from multiple paths (openFolder, restoreFolder, refreshTree). When `S.dirHandle` is null (no folder open), no indexing can legitimately be in progress. Adding a guard at the top of the function is the minimal, complete fix.

The `#idxProgWrap` element starts `display:none` in HTML (`views/app.njk:484`) — this is correct. The bug is that some code path calls `updateIndexProgressState` with a non-idle state before a folder is confirmed open.
