# Phase 39 Plan 01: Auth Gate + JS Cleanup + DEFAULT_APPEARANCE

## Objective

Add a server-side auth gate for `/settings` that redirects unauthenticated users to `/app?login=1`. Remove the now-dead `showSettingsAuthWarning()` function and its call site. Change `DEFAULT_APPEARANCE.theme` from `"light"` to `"system"`.

## Must-Haves

- Unauthenticated GET /settings → redirect to /app?login=1 (not a 401 JSON, not a 404)
- Authenticated GET /settings → settings page renders normally
- `showSettingsAuthWarning` no longer exists in assets/settings.js
- `DEFAULT_APPEARANCE.theme` is `"system"` not `"light"`

## Task 39-01-01: Add /settings auth gate in server.js

<task id="39-01-01">

<objective>
Add a dedicated `app.get('/settings', ...)` route in src/server.js that validates the auth session and redirects unauthenticated users to /app?login=1 before the VIEW_ROUTE_MAP middleware fires.
</objective>

<read_first>
- src/server.js (lines 223-238 — existing route/middleware registration order; lines 259-290 — require statements and router mounts)
- src/core/auth.js (lines 309-363 — resolveAuthUserFromRequest; lines 522-560 — module.exports to confirm export name)
</read_first>

<action>
1. Add a new require at the top of the route-require block in server.js (after the existing `const { csrfProtection, generateToken } = require('./middleware/csrf');` block and before the router requires):

```js
const { resolveAuthUserFromRequest } = require('./core/auth');
```

2. Add a dedicated GET route for /settings immediately BEFORE the VIEW_ROUTE_MAP `app.use(...)` middleware (currently at line 229). Place it directly after the `/` app.get handler (currently lines 223-227):

```js
app.get('/settings', async (req, res, next) => {
  const resolved = await resolveAuthUserFromRequest(req);
  if (!resolved) {
    return res.redirect('/app?login=1');
  }
  try {
    await sendHtmlWithHashes(res, 'settings.njk');
  } catch (err) { next(err); }
});
```

The route must appear BEFORE the `app.use(async (req, res, next) => { ... VIEW_ROUTE_MAP ... })` middleware so Express matches /settings here first.
</action>

<acceptance_criteria>
- `grep "resolveAuthUserFromRequest" src/server.js` returns at least 1 match (require + usage)
- `grep "res.redirect('/app?login=1')" src/server.js` returns 1 match
- `grep "app.get('/settings'" src/server.js` returns 1 match
- `grep "sendHtmlWithHashes(res, 'settings.njk')" src/server.js` returns 1 match (in the /settings route, not just the VIEW_ROUTE_MAP path)
- The /settings route registration appears before the VIEW_ROUTE_MAP `app.use(async (req, res, next)` in the file
</acceptance_criteria>

</task>
