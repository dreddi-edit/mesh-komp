# Phase 39 Plan 02: settings.njk — Theme Default (inline script + select)

## Objective

Change the flash-prevention inline script in `views/settings.njk` to default to `'system'` instead of `'light'` for users with no saved preference. Change the `<option selected>` in the theme `<select>` to `Follow system`.

## Must-Haves

- `views/settings.njk` inline script: fallback is `'system'`, not `'light'`
- Theme `<select>`: `value="system"` option is `selected`, not `value="light"`
- On first load with no localStorage, dark OS → `data-theme="dark"` on `<html>`; light OS → `data-theme="light"`

---

## Task 39-02-01: Change inline theme script fallback from 'light' to 'system'

<task id="39-02-01">

<objective>
Update the flash-prevention inline `<script>` at the top of views/settings.njk (line 23) to fall back to `'system'` instead of `'light'` when no theme preference is stored in localStorage.
</objective>

<read_first>
- views/settings.njk (lines 10-27 — the inline script block)
</read_first>

<action>
On line 23, change the inline script from:
```js
<script>(function(){try{var a=JSON.parse(localStorage.getItem('meshAppearance')||'{}');var t=String(a.theme||'light').trim();if(t==='system')t=window.matchMedia&&window.matchMedia('(prefers-color-scheme:dark)').matches?'dark':'light';document.documentElement.dataset.theme=t==='dark'?'dark':'light';}catch(e){document.documentElement.dataset.theme='light';}}());</script>
```
to:
```js
<script>(function(){try{var a=JSON.parse(localStorage.getItem('meshAppearance')||'{}');var t=String(a.theme||'system').trim();if(t==='system')t=window.matchMedia&&window.matchMedia('(prefers-color-scheme:dark)').matches?'dark':'light';document.documentElement.dataset.theme=t==='dark'?'dark':'light';}catch(e){document.documentElement.dataset.theme='light';}})();</script>
```

The only changes:
1. `a.theme||'light'` → `a.theme||'system'` (the fallback when no theme is stored)
2. `}())` → `})()` (normalized IIFE syntax — functionally equivalent)

The catch block retains `'light'` as the absolute last fallback (DOM error branch) — this is intentional for resilience.
</action>

<acceptance_criteria>
- `grep "||'system'" views/settings.njk` returns 1 match (the inline script)
- `grep "||'light'" views/settings.njk` returns 0 matches
- `grep "prefers-color-scheme:dark" views/settings.njk` returns 1 match (the system resolution logic is still present)
- `grep "dataset.theme" views/settings.njk` returns at least 1 match
</acceptance_criteria>

</task>

## Task 39-02-02: Change theme <select> default from Light to Follow system

<task id="39-02-02">

<objective>
Update the theme `<select>` in views/settings.njk so that `Follow system` (value="system") is the `selected` option instead of `Light` (value="light").
</objective>

<read_first>
- views/settings.njk (lines 900-910 — the theme select field)
</read_first>

<action>
In the theme `<select>` (around line 904-908), change:
```html
<select class="field-select" id="app-theme" name="theme">
  <option value="light" selected>Light</option>
  <option value="dark">Dark</option>
  <option value="system">Follow system</option>
</select>
```
to:
```html
<select class="field-select" id="app-theme" name="theme">
  <option value="light">Light</option>
  <option value="dark">Dark</option>
  <option value="system" selected>Follow system</option>
</select>
```

Only the `selected` attribute moves from `value="light"` to `value="system"`. The order and labels of all three options remain unchanged.

Note: In practice, the settings JS populates the form with the stored value on load — the HTML `selected` attribute is the fallback for users with no stored value (first load). This ensures new users see "Follow system" as the pre-selected default.
</action>

<acceptance_criteria>
- `grep 'value="system" selected' views/settings.njk` returns 1 match
- `grep 'value="light" selected' views/settings.njk` returns 0 matches
- `grep 'value="light">' views/settings.njk` returns 1 match (Light option still exists, just not selected)
- `grep 'Follow system' views/settings.njk` returns 1 match
</acceptance_criteria>

</task>
