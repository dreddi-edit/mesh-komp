# Plan 20-02: CSRF Token Protection

<objective>
Add CSRF token protection using csrf-csrf double-submit cookie pattern to all mutating routes. Upgrade from Origin/Referer check to token-based CSRF.
</objective>

<tasks>

<task id="1" name="Install csrf-csrf">
<read_first>
- package.json
</read_first>
<action>
Run `npm install csrf-csrf` to add the double-submit cookie CSRF package.
</action>
<acceptance_criteria>
- `grep '"csrf-csrf"' package.json` returns a match
</acceptance_criteria>
</task>

<task id="2" name="Create CSRF middleware">
<read_first>
- src/server.js
- src/config/index.js
</read_first>
<action>
Create `src/middleware/csrf.js`:

1. Import `doubleCsrf` from `csrf-csrf`
2. Configure with:
   - `getSecret: () => config.CSRF_SECRET` (add `CSRF_SECRET` to config, fallback to `JWT_SECRET`)
   - `cookieName: '_csrf'`
   - `cookieOptions: { httpOnly: true, sameSite: 'strict', secure: config.NODE_ENV === 'production' }`
   - `getTokenFromRequest: (req) => req.headers['x-csrf-token']`
3. Export `{ csrfProtection, generateToken }` from `doubleCsrf()`

Add `CSRF_SECRET` to `src/config/index.js` with fallback to `JWT_SECRET`.
</action>
<acceptance_criteria>
- File `src/middleware/csrf.js` exists
- `grep "doubleCsrf" src/middleware/csrf.js` matches
- `grep "CSRF_SECRET" src/config/index.js` matches
</acceptance_criteria>
</task>

<task id="3" name="Wire CSRF into server and add token endpoint">
<read_first>
- src/server.js
</read_first>
<action>
In `src/server.js`:

1. Import `{ csrfProtection, generateToken }` from `./middleware/csrf`
2. Remove the old Origin/Referer CSRF check (approx lines 59-88)
3. Add a GET endpoint `/api/csrf-token` that returns `{ ok: true, token: generateToken(req, res) }`
4. Apply `csrfProtection` middleware to all POST/PUT/PATCH/DELETE routes — mount it after auth middleware but before route handlers

Implementation: add `csrfProtection` as Express middleware for mutating methods only:
```javascript
app.use((req, res, next) => {
  if (['POST', 'PUT', 'PATCH', 'DELETE'].includes(req.method)) {
    return csrfProtection(req, res, next);
  }
  next();
});
```
</action>
<acceptance_criteria>
- `grep "csrfProtection" src/server.js` matches
- `grep "/api/csrf-token" src/server.js` matches
- Old Origin/Referer CSRF block removed from server.js
- `npm test` passes
</acceptance_criteria>
</task>

<task id="4" name="Update frontend to include CSRF token">
<read_first>
- assets/mesh-client.js
</read_first>
<action>
In `assets/mesh-client.js`:

1. Add a function to fetch CSRF token: fetch `/api/csrf-token`, cache the token
2. Before every POST/PUT/PATCH/DELETE fetch call, include `X-CSRF-Token` header with the cached token
3. On 403 response with CSRF error, auto-refresh token and retry once

Also search other `assets/*.js` files for direct `fetch()` calls that POST/PUT/PATCH/DELETE and ensure they go through the CSRF-aware client or include the header.
</action>
<acceptance_criteria>
- `grep "X-CSRF-Token" assets/mesh-client.js` matches
- `grep "csrf-token" assets/mesh-client.js` matches
- `npm test` passes
</acceptance_criteria>
</task>

</tasks>
