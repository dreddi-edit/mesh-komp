# Plan 28-01: Fix Workspace Indexing Stall at ~55%

## Objective

Fix the workspace indexing pipeline that stalls at approximately 55% progress. The root cause is `deepScanAll` scanning ALL directories including `node_modules`, causing `discoveredUnits` to explode and the progress ratio to plateau. Additionally, the sync pipeline may hang when tree-sitter workers are saturated.

## must_haves

1. Workspace indexing completes to 100% for any project (including large Node.js repos with node_modules)
2. Progress bar advances smoothly without stalling
3. File explorer still shows ALL files including node_modules (transparency preserved)
4. Indexing only processes files that pass `isIndexableWorkspacePath`

<action>
In `assets/app-workspace.js`, modify the `deepScanAll` function (around line 924) to skip directories that match `INDEX_SKIP_DIRS` pattern during the recursive deep scan. The key change is adding a check at the start of the directory processing block:

```javascript
async function deepScanAll(items, progress = null){
  const tracker = progress || createDeepScanProgress(items);
  paintDeepScanProgress(tracker, { force: true });
  for(const item of items){
    if(!item) continue;
    tracker.visitedUnits += 1;
    if(item.isDir){
      // Skip known-heavy directories for indexing scan — file explorer still shows them
      if(INDEX_SKIP_DIRS.test(item.path)){
        paintDeepScanProgress(tracker);
        continue;
      }
      await ensureChildren(item);
      const childCount = Array.isArray(item.children) ? item.children.length : 0;
      tracker.discoveredUnits += childCount;
      if(item.children){
        for(const child of item.children){
          if(!child.isDir && child.path && isIndexableWorkspacePath(child.path) && !S.compressionMap.has(child.path)){
            S.compressionMap.set(child.path,{path:child.path,rawBytes:child.size||0,capsuleBytes:0,status:'pending'});
          }
        }
      }
    }
    paintDeepScanProgress(tracker);
    if(item.isDir && item.children && !INDEX_SKIP_DIRS.test(item.path)) await deepScanAll(item.children, tracker);
  }
  paintDeepScanProgress(tracker, { force: true });
  return tracker;
}
```

Critical: The `INDEX_SKIP_DIRS` regex tests against the full path (e.g., `src/node_modules/foo`), not just the directory name. The existing regex `/(^|\/)(node_modules|\.git|dist|build|\.next|__pycache__)(\/|$)/i` already handles this correctly since it matches the segment anywhere in the path.

Note: `fullScan` (the file explorer scan) must NOT be changed — it intentionally scans everything for explorer transparency.
</action>

</acceptance_criteria>
