# Plan 28-04: Fix Terminal to Connect Locally When Server Runs on Localhost

## Objective

The terminal currently always connects via WebSocket to the server that served the page. When that server is EC2, the terminal runs on EC2. When running locally, it correctly runs locally. The fix is to:
1. Ensure local-path workspaces use the local filesystem root as CWD (this already works)
2. When server detects it's running locally, use the workspace's actual rootPath as CWD
3. Add clear UX indication of whether the terminal is local or remote
4. When the user opens a local folder via File System Access API, pass the folder path to the terminal so it starts in the right directory

## must_haves

1. Terminal opens in the workspace root directory when a local folder is open
2. Terminal clearly indicates whether it's a local or remote session
3. Workspace folder path is passed to the terminal WebSocket connection

<action>
The terminal WebSocket connects to `/terminal` without passing the current workspace info. The backend's `resolveTerminalCwd()` reads `core.localAssistantWorkspace` which may not have `rootPath` set when using the File System Access API (browser-opened folders set `sourceKind` to `'upload'`, not `'local-path'`).

Fix 1: In `assets/app-workspace.js`, pass the directory name as a query parameter when connecting:

```javascript
// Around line 1412, modify the WebSocket URL:
const wsUrl = new URL(prot + '//' + tHost + '/terminal');
if (S.dirName) wsUrl.searchParams.set('folder', S.dirName);
if (S.workspaceId) wsUrl.searchParams.set('workspaceId', S.workspaceId);
const ws = new WebSocket(wsUrl.toString());
S.termWs = ws;
```

Fix 2: In `src/routes/terminal.routes.js`, read the folder/workspaceId from the URL and use it to help resolve CWD. Update the `wss.on('connection')` handler around line 250:

```javascript
const urlParams = new URL(req.url, 'http://localhost').searchParams;
const shellPref = urlParams.get('shell');
const clientFolder = urlParams.get('folder') || '';
const clientWorkspaceId = urlParams.get('workspaceId') || '';
```

Then pass these to `resolveTerminalCwd`:

```javascript
cwdInfo = await resolveTerminalCwd(projectRoot, {
  localAssistantWorkspace: core.localAssistantWorkspace,
  clientFolder,
  clientWorkspaceId,
  ...materialDeps,
});
```

Fix 3: Update `resolveTerminalCwd` to also check if the local workspace matches the client's folder even when sourceKind isn't 'local-path':

```javascript
async function resolveTerminalCwd(projectRoot, deps = {}) {
  const workspace = deps.localAssistantWorkspace || {};

  // Priority 1: Local-path workspace with real rootPath
  if (String(workspace.sourceKind || '').trim() === 'local-path' && workspace.rootPath) {
    return { cwd: workspace.rootPath, note: workspace.rootPath };
  }

  // Priority 2: If workspace has a rootPath (set by openLocalWorkspaceWithFallback)
  if (workspace.rootPath && workspace.rootPath !== '') {
    return { cwd: workspace.rootPath, note: workspace.rootPath };
  }

  // Priority 3: Upload workspace — materialize to temp dir
  if (String(workspace.workspaceId || '').trim() && String(workspace.folderName || '').trim()) {
    const cwd = await materializeUploadWorkspaceRoot(workspace, deps);
    return { cwd, note: `${cwd} (materialized from uploaded workspace)` };
  }

  return { cwd: projectRoot, note: projectRoot };
}
```
