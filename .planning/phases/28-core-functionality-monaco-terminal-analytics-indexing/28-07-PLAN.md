# Plan 28-07: Settings Persistence Hardening

## Objective

Settings persistence infrastructure is complete (localStorage + DynamoDB via `/api/user/store`). Two gaps exist:

1. **`initAppearance()` save handler** uses `saveJSON` (fire-and-forget, errors silently swallowed) instead of `persistJSON` (awaited, shows error toast). If the session has expired, the user clicks "Save preferences", sees "Saved" toast, but settings are only in localStorage — lost on another device or after clearing storage.

2. **`preloadUserStoreCache()` 401 handling**: If the server returns 401 (session missing), the catch block silently ignores it and falls back to localStorage values. The user sees their settings loaded but doesn't know they're unauthenticated. Any subsequent save attempt will also fail silently.

Fix both: make appearance save use `persistJSON`, and surface a visible warning banner when `preloadUserStoreCache` detects an auth failure.

## must_haves

1. Clicking "Save preferences" in Appearance settings awaits the server save; shows an error toast if the PUT request fails
2. If the settings page loads but the session is invalid (401 on preload), a visible warning is shown: "Not signed in — settings changes won't be saved to your account"
3. No regression in existing save flows (form submits via `initForms` already use `persistJSON` — do not change those)
4. Settings still load and display correctly even when unauthenticated (localStorage fallback preserved)

```javascript
    form.addEventListener("submit", e => {
      e.preventDefault();
      const data = Object.fromEntries(new FormData(form));
      saveJSON("meshAppearance", data);
      updatePreviews(data);
      applySettingsTheme();
      showToast("Saved", "Appearance preferences saved.");

```

Replace with:
```javascript
    form.addEventListener("submit", async e => {
      e.preventDefault();
      const data = Object.fromEntries(new FormData(form));
      const submit = form.querySelector("button[type='submit']");
      withButtonBusy(submit, true, "Saving...");
      try {
        await persistJSON("meshAppearance", data);
        updatePreviews(data);
        applySettingsTheme();
        showToast("Saved", "Appearance preferences saved.");
      } catch (error) {
        showToast("Save failed", error.message || "Could not save appearance settings.");
      } finally {
        withButtonBusy(submit, false);
      }
    });
```

Note: `withButtonBusy` is already defined at line ~204 in settings.js. `persistJSON` is already defined at line ~134.

  try {
    const query = encodeURIComponent(SAFE_USER_STORE_KEYS.join(","));
    const response = await fetch(`/api/user/store?keys=${query}`, { credentials: "same-origin" });
    if (response.status === 401) {
      showSettingsAuthWarning();

      const body = await response.json().catch(() => ({}));
      const remote = body?.data && typeof body.data === "object" ? body.data : {};
      for (const key of SAFE_USER_STORE_KEYS) {
        if (remote[key] !== undefined) USER_STORE_CACHE.set(key, remote[key]);
      }
    }
  } catch {}
