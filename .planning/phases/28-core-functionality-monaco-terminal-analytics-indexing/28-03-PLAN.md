# Plan 28-03: Wire Real Data into Operations Panel and Compression Analytics

## Objective

Make the operations panel and compression analytics display real system data instead of empty/placeholder state. The operations store initializes empty and never auto-populates. Compression analytics is blocked by the indexing stall (fixed in Plan 01). This plan adds real operational telemetry and ensures compression data flows correctly.

## must_haves

1. Operations panel shows real log entries from server lifecycle events
2. Compression analytics shows real rawBytes and capsuleBytes for indexed files
3. Data updates live without requiring page refresh

Additionally, ensure `loadCompressionMap()` (the API fetch path) correctly populates the map. Read the function around line 1460:

```javascript
async function loadCompressionMap() {
  try {
    const resp = await api('/api/app/compression');
    if (resp?.ok && Array.isArray(resp.files)) {
      for (const meta of resp.files) {
        if (!meta?.path) continue;
        S.compressionMap.set(meta.path, {
          path: meta.path,
          rawBytes: meta.rawBytes || meta.originalSize || 0,
          capsuleBytes: meta.capsuleBytes || meta.compressedSize || 0,
          status: meta.status || 'completed',
        });
      }
    }
  } catch(e) { console.error('[compression] load failed', e); }
}
```

But wait — check the field name. The frontend `loadCompressionMap` looks for `rawBytes` and `capsuleBytes`, but `buildWorkspaceFileListingEntry` returns `rawBytes` as a direct field. Let me trace the exact mapping:

Backend returns: `{ path, originalSize, compressedSize, indexed, kind, ... }`

The `rawBytes` and `capsuleBytes` fields are NOT in the `buildWorkspaceFileListingEntry` return. They're missing! Fix this:

In `src/core/index.js`, update `buildWorkspaceFileListingEntry` to include the fields the frontend expects:

```javascript
function buildWorkspaceFileListingEntry(meta) {
  const rawBytes = Number(meta?.rawStorage?.rawBytes || meta?.originalSize || 0);
  const capsuleBytes = Number(meta?.capsuleCache?.capsule?.capsuleBytes || 0);
  const compressionRatio = rawBytes > 0 ? Math.max(0, 1 - capsuleBytes / rawBytes) : 0;
  return {
    path: meta.path,
    rawBytes,
    capsuleBytes,
    compressionRatio,
    originalSize: Number(meta.originalSize || 0),
    compressedSize: Number(meta.compressedSize || 0),
    indexed: workspaceRecordIndexed(meta),
    kind: meta?.kind || (workspaceRecordIndexed(meta) ? 'source' : 'pending'),
    fileType: String(meta?.fileType || ''),
    parserFamily: String(meta?.parserFamily || ''),
    parseOk: Boolean(meta?.parseOk),
    capsuleMode: String(meta?.capsuleMode || ''),
    status: String(meta?.status || (workspaceRecordIndexed(meta) ? 'completed' : 'pending')),
    error: String(meta?.error || ''),
  };
}
```

Then verify the frontend `loadCompressionMap` reads these fields correctly.
