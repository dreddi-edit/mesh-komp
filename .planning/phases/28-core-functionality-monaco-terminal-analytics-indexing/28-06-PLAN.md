# Plan 28-06: Settings UI Visual Alignment

## Objective

Settings pages share `tokens.css` with the app but have component-level visual gaps. The most visible discrepancy is `.btn-primary` in settings uses `background: var(--text-hi)` (near-white in dark mode) instead of the accent blue used across the app (`var(--accent)` = `#0098ff` dark / `#005fb8` light). Additional gaps: button border-radius uses `--r-sm` (6px) vs app's 4px; form inputs use `.field` class not `.field-input`; overall panel card border-radius uses `--r` (10px) vs app's 4px. 

Align these component atoms to match the app's design language while keeping the settings layout intact.

## must_haves

1. `.btn-primary` in settings uses the accent color (`var(--accent)`) matching the app's primary action buttons
2. Settings pages use the same button font size and padding as the app's action buttons
3. Visual inspection shows settings "Save" buttons match the blue accent color seen in the app's toolbar buttons
4. No layout shift or broken elements after the CSS changes

## Tasks

<task id="1">
<title>Fix .btn-primary to use accent color matching the app</title>
<read_first>
- assets/mesh-settings.css (lines 147-162, button definitions)
- assets/app-workspace.css (lines 78, 149 — .auth-btn and .scm-btn use var(--ac) background)
- assets/tokens.css (accent token definitions)
</read_first>
<action>
In assets/mesh-settings.css, change the `.btn-primary` background from `var(--text-hi)` to `var(--accent)`.

Find this block (lines ~155-156):
```css
.btn-primary   { background: var(--text-hi); color: #fff; }
.btn-primary:hover { opacity: 0.86; }
```

Replace with:
```css
.btn-primary   { background: var(--accent); color: #fff; }
.btn-primary:hover { background: var(--accent-2); opacity: 1; }
```

Also ensure `.btn-primary` has a consistent hover state. The app uses `--ac2` for hover (`#007acc` dark / `#004e99` light) — settings calls this `--accent-2`. Add `--accent-2` to the settings CSS variable aliases if not present.

Check if `--accent-2` is defined in the `:root` block of mesh-settings.css. It is (line ~24): `--accent-2: var(--color-accent-secondary)`. No change needed to the variable definition.
</action>
<acceptance_criteria>
- grep '.btn-primary' assets/mesh-settings.css shows `background: var(--accent)` not `background: var(--text-hi)`
- grep 'btn-primary:hover' assets/mesh-settings.css shows `background: var(--accent-2)` or equivalent accent hover state
</acceptance_criteria>
</task>
