# Plan 21-01: XSS Audit + DOMPurify Setup

<objective>
Audit all unsafe DOM injection usage across frontend JS, classify by risk, and create a shared safe-DOM utility module with DOMPurify integration for consistent XSS-safe rendering.
</objective>

<tasks>

<task id="1" name="Audit unsafe DOM injection usage">
<read_first>
- assets/app-workspace.js
- assets/app.js
- assets/app-graph.js
- assets/settings.js
</read_first>
<action>
Search all assets JS files for unsafe DOM injection patterns: setting element content via raw string assignment, insertAdjacentHTML, etc.

For each site, classify as:
- HIGH RISK: user-supplied content (chat messages, file names, user input, git messages, search results)
- MEDIUM RISK: content from API responses that could contain user data
- LOW RISK: static HTML template construction with no user data

Create a tracking summary at the top of a new file `assets/dom-utils.js` listing the audit results (file:line, risk level). This drives the remediation tasks.
</action>
<acceptance_criteria>
- File `assets/dom-utils.js` exists with audit summary
- Audit covers all assets JS files
</acceptance_criteria>
</task>

<task id="2" name="Create safe DOM utility module">
<read_first>
- assets/app-workspace.js
</read_first>
<action>
Create `assets/dom-utils.js` with shared safe-rendering helpers:

1. `safeEl(tag, text, className)` - creates an element using createElement + textContent (XSS-safe)
2. `sanitizeHtml(html)` - wraps DOMPurify.sanitize() with an allowlist of safe tags (b, i, em, strong, a, code, pre, span, div, p, br, ul, ol, li, h1-h6, table, tr, td, th, img, blockquote, hr, details, summary) and safe attributes (href, class, target, rel, src, alt, title, id)
3. `safeHtml(el, html)` - sets element content using sanitizeHtml output

Ensure DOMPurify script tag is included in HTML pages that render dynamic content.
</action>
<acceptance_criteria>
- File `assets/dom-utils.js` exists with `safeEl`, `sanitizeHtml`, `safeHtml` functions
- `grep "DOMPurify" assets/dom-utils.js` matches
- `grep "textContent" assets/dom-utils.js` matches
</acceptance_criteria>
</task>

</tasks>

<verification>
- assets/dom-utils.js contains safe DOM helpers
- DOMPurify is available in HTML pages that render user content
</verification>

<must_haves>
- Complete audit of unsafe DOM injection across all frontend JS
- Safe DOM utility module with DOMPurify integration
</must_haves>
