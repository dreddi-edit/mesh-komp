# Plan 08-02: Dependency Graph Animations and Live Updates

## Objective

Improve the dependency graph UX with: (1) entrance animations when nodes/edges appear,
(2) smooth cross-fade on rebuild instead of jarring flash, (3) debounced live update
when files change in the background.

## Background from Research

Current state — already implemented in the working tree:
- **Cross-fade on rebuild**: Lines 381-387 in app-graph.js already fade out the old SVG
  before replacing. This is working.
- **Live update debounce**: Lines 748-752 already listen to `mesh-indexing-background-progress`
  with a 1500ms debounce. This is working.
- **CSS entrance fade**: Lines 644-650 set `opacity: 0` and `transition: opacity 0.35s ease`
  then trigger via `requestAnimationFrame`. This is working.

**What's still missing / improvable:**
1. The entrance transition only fades opacity — nodes appear at their final positions with
   no spatial entrance. Adding a subtle scale animation (0.8 → 1.0) makes the entrance
   feel more alive without anime.js (which has v4 API incompatibilities).
2. The cross-fade waits 190ms blocking the `initWorkspaceGraph` async function — this
   works but could be made non-blocking with a CSS class approach for future builds.
3. The stagger between nodes is missing — all nodes fade in at once. Adding a CSS
   animation-delay stagger makes large graphs feel more dynamic.

  <action>
  In `assets/app-graph.js`, replace the current entrance animation block (lines ~644-650):

  Current code:
  ```js
  // Entrance fade via CSS — avoids anime.js v4 API incompatibilities
  node.style('opacity', '0').style('transition', 'opacity 0.35s ease');
  link.style('opacity', '0').style('transition', 'opacity 0.25s ease');
  requestAnimationFrame(() => {
      node.style('opacity', '1');
      setTimeout(() => link.style('opacity', '0.4'), 80);
  });
  ```

  Replace with:
  ```js
  // Entrance animation: scale + fade with per-node stagger
  // Uses CSS transitions only — avoids anime.js v4 API incompatibilities.
  node.style('opacity', '0')
      .style('transform', d => `translate(${d.x},${d.y}) scale(0.6)`)
      .style('transition', 'opacity 0.35s ease, transform 0.4s cubic-bezier(0.34,1.56,0.64,1)');
  link.style('opacity', '0').style('transition', 'opacity 0.3s ease');

  // Stagger entrance: nodes appear in waves of 6 with 18ms delay between waves
  node.nodes().forEach((el, i) => {
      const delay = Math.floor(i / 6) * 18;
      el.style.transitionDelay = delay + 'ms';
  });

  requestAnimationFrame(() => {
      node.style('opacity', '1').style('transform', null);
      setTimeout(() => {
          link.style('opacity', '0.4');
          // Clear delays after animation completes so drag/hover transitions aren't delayed
          setTimeout(() => node.style('transition-delay', '0ms'), 600);
      }, 80);
  });
  ```

  Note: `style('transform', null)` removes the inline transform override so D3's `tick`
  handler (which sets `transform` via `.attr('transform', ...)`) takes back control.
  The `attr` vs `style` distinction matters: D3 uses `.attr('transform')` for the tick
  animation while CSS transitions use `.style('transform')` — setting style to null
  defers back to the attr value.

  Wait — this approach has a conflict: D3's tick handler uses `.attr('transform', ...)` not
  `.style('transform', ...)`. SVG `transform` attribute and CSS `transform` property are
  separate. The CSS transform at scale(0.6) is applied on top of the SVG attr transform.
  Setting `.style('transform', null)` removes the CSS override correctly, leaving the attr
  transform from the tick handler intact.

  Actually for correctness, use `translateX`/`translateY` CSS to avoid conflict:

  ```js
  // Entrance animation: fade + scale via CSS (no anime.js dependency)
  node.style('opacity', '0')
      .style('transition', 'opacity 0.4s ease');
  link.style('opacity', '0').style('transition', 'opacity 0.3s ease');

  // Apply stagger delays: groups of 8 nodes share a delay tier, 20ms between tiers
  node.nodes().forEach((el, i) => {
      el.style.transitionDelay = (Math.floor(i / 8) * 20) + 'ms';
  });

  requestAnimationFrame(() => {
      node.style('opacity', '1');
      setTimeout(() => {
          link.style('opacity', '0.4');
          // Remove stagger delays after animation so hover/drag transitions are instant
          setTimeout(() => node.nodes().forEach(el => { el.style.transitionDelay = '0ms'; }), 700);
      }, 80);
  });
  ```
  </action>

  <acceptance_criteria>

    - `assets/app-graph.js` contains `transitionDelay` being applied to node elements
    - The `node.nodes().forEach` loop setting `transitionDelay` appears before the `requestAnimationFrame` call
    - After the animation, `transitionDelay` is reset to `'0ms'` via a setTimeout
  </acceptance_criteria>
