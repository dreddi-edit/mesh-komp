<objective>
Replace the CDN Monaco loader and Blob worker pattern in app.njk and the polling initMonaco() in app-workspace.js with a self-hosted, synchronous, data:-URL-based implementation.

Purpose: Eliminates race conditions (defer + setInterval polling) and CDN dependency. The synchronous loader.js guarantees require() is defined before any deferred script runs, making initMonaco() a single require.config() + require() call.
Output: app.njk with self-hosted loader (no defer), data: URL worker config, Monaco CSS link. app-workspace.js with polling-free initMonaco().
</objective>

  <files>assets/app-workspace.js</files>

  <action>
    In assets/app-workspace.js, find the `initMonaco` function starting at approximately line 1138 and replace the entire function body.
    
    Replace this entire function:
    ```javascript
    function initMonaco(cb) {
      const VS_BASE = 'https://cdn.jsdelivr.net/npm/monaco-editor@0.52.2/min/vs';
      const POLL_INTERVAL_MS = 50;
      const MAX_WAIT_MS = 8000;
      let elapsed = 0;
      const tid = setInterval(() => {
        elapsed += POLL_INTERVAL_MS;
        if (typeof require !== 'undefined') {
          clearInterval(tid);
          require.config({ paths: { vs: VS_BASE } });
          require(['vs/editor/editor.main'], () => { S.monacoReady = true; cb(); });
        } else if (elapsed >= MAX_WAIT_MS) {
          clearInterval(tid);
        }
      }, POLL_INTERVAL_MS);
    }
    ```
    
    With this replacement:
    ```javascript
    function initMonaco(cb) {
      require.config({ paths: { vs: '/assets/monaco/vs' } });
      require(['vs/editor/editor.main'], () => { S.monacoReady = true; cb(); });
    }
    ```
    
    Why this works: `loader.js` is now loaded synchronously (no `defer`) in app.njk, so `require` is defined in the global scope before any `defer`-loaded scripts (including app-workspace.js) execute. No polling is needed.
    
    The `vs` path now points to the self-hosted route `/assets/monaco/vs` instead of the CDN.
    
    Do NOT change `createEditor()` or any other Monaco-related code — only `initMonaco()` changes.
  </action>
