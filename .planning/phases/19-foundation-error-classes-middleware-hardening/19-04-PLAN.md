# Plan 19-04: LRU Cache Replacements

<objective>
Replace all unbounded `new Map()` stores with `lru-cache` to prevent heap growth. Rate limiter gets max 5,000 entries. Workspace file Map and other caches get configurable max sizes.
</objective>

<tasks>

<task id="1" name="Add LRU config values">
<read_first>
- src/config/index.js
</read_first>
<action>
In `src/config/index.js` `buildConfig(env)`, add these keys:

```javascript
RATE_LIMITER_MAX_ENTRIES: parseIntegerInRange(env.RATE_LIMITER_MAX_ENTRIES, 1000, 50000) || 5000,
WORKSPACE_FILE_CACHE_MAX: parseIntegerInRange(env.WORKSPACE_FILE_CACHE_MAX, 100, 100000) || 10000,
SESSION_CACHE_MAX: parseIntegerInRange(env.SESSION_CACHE_MAX, 100, 10000) || 1000,
CODEC_SESSION_CACHE_MAX: parseIntegerInRange(env.CODEC_SESSION_CACHE_MAX, 100, 5000) || 500,
ASSISTANT_RUNS_CACHE_MAX: parseIntegerInRange(env.ASSISTANT_RUNS_CACHE_MAX, 50, 5000) || 500,
INFER_FILES_CACHE_MAX: parseIntegerInRange(env.INFER_FILES_CACHE_MAX, 50, 5000) || 500,
```

Use the existing `parseIntegerInRange` helper from `env-utils.js`.
</action>
<acceptance_criteria>
- `grep "RATE_LIMITER_MAX_ENTRIES" src/config/index.js` matches
- `grep "WORKSPACE_FILE_CACHE_MAX" src/config/index.js` matches
- `grep "SESSION_CACHE_MAX" src/config/index.js` matches
</acceptance_criteria>
</task>

<task id="2" name="Replace rate limiter Map with LRU">
<read_first>
- src/middleware/rate-limiter.js
</read_first>
<action>
In `src/middleware/rate-limiter.js`:

1. Add `const { LRUCache } = require('lru-cache');` at top
2. Find the `new Map()` that stores rate limit entries (approx line 45)
3. Replace with: `new LRUCache({ max: config.RATE_LIMITER_MAX_ENTRIES, ttl: windowMs })`

5. Ensure the get/set API matches: `cache.get(key)`, `cache.set(key, value)`
</action>
<acceptance_criteria>
