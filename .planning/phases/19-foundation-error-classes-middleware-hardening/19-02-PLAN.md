# Plan 19-02: Centralized Error Middleware

<objective>
Wire `express-async-errors` and create centralized error-handling middleware that maps AppError subclasses to structured JSON responses. Eliminates per-route try/catch pattern.
</objective>

<tasks>

<task id="1" name="Install express-async-errors">
<read_first>
- package.json
</read_first>
<action>
Run `npm install express-async-errors` to add the package. This package monkey-patches Express route handlers to catch async errors automatically.
</action>
<acceptance_criteria>
- `grep "express-async-errors" package.json` returns a match in dependencies
- `node -e "require('express-async-errors')"` exits without error
</acceptance_criteria>
</task>

<task id="2" name="Create error-handler middleware">
<read_first>
- src/errors/index.js
- src/logger.js
- src/routes/route-utils.js
</read_first>
<action>
Create `src/middleware/error-handler.js`:

1. Import `AppError` from `../errors`
2. Import `logger` from `../logger`
3. Export a 4-argument Express error middleware: `(err, req, res, next)`
4. Logic:
   - If `err instanceof AppError`: use `err.statusCode` and `err.code`
   - Otherwise: statusCode = 500, code = 'INTERNAL_ERROR'
   - Log via `logger.error('Request error', { requestId: req.id, statusCode, code: err.code || 'INTERNAL_ERROR', error: err.message, stack: err.stack })`
   - Return `res.status(statusCode).json({ ok: false, error: err.message })` — matches existing envelope format
   - If `err instanceof ValidationError` and `err.fields`: include `fields` in response
5. Use `'use strict'`, CommonJS, JSDoc

Response format for validation errors:
```json
{ "ok": false, "error": "message", "fields": { "email": "invalid" } }
```

Response format for other errors:
```json
{ "ok": false, "error": "message" }
```
</action>
<acceptance_criteria>
- File `src/middleware/error-handler.js` exists
- `grep "err instanceof AppError" src/middleware/error-handler.js` matches
- `grep "ok: false" src/middleware/error-handler.js` matches
- `grep "logger.error" src/middleware/error-handler.js` matches
- Function has 4 parameters (err, req, res, next)
</acceptance_criteria>
</task>

<task id="3" name="Wire into server.js">
<read_first>
- src/server.js
</read_first>
<action>
In `src/server.js`:

1. Add `require('express-async-errors')` near the top of the file (before Express app creation)
2. Import the error handler: `const { errorHandler } = require('./middleware/error-handler')`
3. Mount error handler AFTER all routes (must be the last middleware): `app.use(errorHandler)`
4. Do NOT remove existing `safeRouteError()` usage yet — Phase 19 adopts incrementally per D-03
</action>
<acceptance_criteria>

- `grep "errorHandler" src/server.js` matches
- `npm test` passes with 0 failures
</acceptance_criteria>
</task>
