# Plan 19-01: Error Class Hierarchy

<objective>
Create typed error class hierarchy under `src/errors/` with base `AppError` and subclasses: `ValidationError`, `NotFoundError`, `AuthError`, `ConflictError`. Machine-readable error codes for downstream CloudWatch metric filters (Phase 27).
</objective>

<tasks>

<task id="1" name="Create error class module">
<read_first>
- src/routes/route-utils.js
- src/logger.js
</read_first>
<action>
Create `src/errors/index.js` with the following classes:

1. `AppError` extends `Error`:
   - Constructor params: `code` (string), `statusCode` (number), `message` (string), `cause` (optional Error)
   - Sets `this.name = 'AppError'`, `this.code = code`, `this.statusCode = statusCode`
   - If `cause` provided, sets `this.cause = cause`

2. `ValidationError` extends `AppError`:
   - Constructor params: `message` (string), `fields` (object, optional), `cause` (optional)
   - Calls super with code `'VALIDATION_ERROR'`, statusCode `400`
   - Sets `this.fields = fields || {}`

3. `NotFoundError` extends `AppError`:
   - Constructor params: `resource` (string), `id` (string, optional), `cause` (optional)
   - Calls super with code `'NOT_FOUND'`, statusCode `404`, message `${resource}${id ? ` ${id}` : ''} not found`

4. `AuthError` extends `AppError`:
   - Constructor params: `message` (string, default `'Authentication required'`), `cause` (optional)
   - Calls super with code `'AUTH_ERROR'`, statusCode `401`

5. `ConflictError` extends `AppError`:
   - Constructor params: `message` (string), `cause` (optional)
   - Calls super with code `'CONFLICT'`, statusCode `409`

Use `'use strict'` at top. CommonJS `module.exports`. JSDoc on each class.
</action>
<acceptance_criteria>
