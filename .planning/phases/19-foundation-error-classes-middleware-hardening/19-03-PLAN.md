# Plan 19-03: Helmet + CORS Middleware

<objective>
Replace hand-rolled security headers with helmet package and wire CORS with explicit origin allowlist. CSP nonces NOT enforced yet — keep `unsafe-inline` for Phase 20.
</objective>

<task id="2" name="Add CORS origins to config">
<read_first>
- src/config/index.js
</read_first>
<action>
In `src/config/index.js`, add to the config object built by `buildConfig(env)`:

```javascript
CORS_ORIGINS: env.CORS_ORIGINS
  ? env.CORS_ORIGINS.split(',').map(s => s.trim())
  : (env.NODE_ENV === 'production' ? [] : ['http://localhost:3000', 'http://localhost:3001']),
```

Also add validation in `validateConfig()`: if `NODE_ENV === 'production'` and `CORS_ORIGINS` is empty array, add a warning: `'CORS_ORIGINS not set — CORS will reject all cross-origin requests in production'`.
</action>
<acceptance_criteria>
- `grep "CORS_ORIGINS" src/config/index.js` returns matches
- `grep "localhost:3000" src/config/index.js` returns a match (dev default)
</acceptance_criteria>
</task>

<task id="3" name="Replace security headers with helmet and wire CORS">
<read_first>
- src/server.js
</read_first>
<action>
In `src/server.js`:

1. Add requires at top:
   ```javascript
   const helmet = require('helmet');
   const cors = require('cors');
   ```

2. Remove the hand-rolled security headers block (approximately lines 31-57 that set X-Content-Type-Options, X-Frame-Options, CSP, etc.)

3. Replace with helmet configuration:
   ```javascript
   app.use(helmet({
     contentSecurityPolicy: {
       directives: {
         defaultSrc: ["'self'"],
         scriptSrc: ["'self'", "'unsafe-inline'"],
         styleSrc: ["'self'", "'unsafe-inline'"],
         imgSrc: ["'self'", "data:", "blob:"],
         connectSrc: ["'self'", "wss:", "ws:"],
         fontSrc: ["'self'"],
         objectSrc: ["'none'"],
         frameAncestors: ["'none'"],
       },
     },
     crossOriginEmbedderPolicy: false,
   }));
   ```
   Note: `unsafe-inline` kept intentionally — Phase 20 removes it with nonces.

4. Wire CORS after helmet:
   ```javascript
   app.use(cors({
     origin: config.CORS_ORIGINS,
     credentials: true,
     methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
     allowedHeaders: ['Content-Type', 'Authorization', 'X-CSRF-Token'],
   }));
   ```

5. The `cors` package is already in package.json but not imported — just add the require and usage.
</action>
<acceptance_criteria>
- `grep "require('helmet')" src/server.js` matches
- `grep "require('cors')" src/server.js` matches
- `grep "app.use(helmet" src/server.js` matches
- `grep "app.use(cors" src/server.js` matches
- Hand-rolled `X-Content-Type-Options` or `X-Frame-Options` no longer set manually in server.js
- `npm test` passes with 0 failures
</acceptance_criteria>
</task>
