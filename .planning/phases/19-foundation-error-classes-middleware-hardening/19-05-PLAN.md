# Plan 19-05: HTTP Cache Headers + Directory Cleanup

<objective>
Add Cache-Control headers to read-only GET API endpoints and resolve empty directories.
</objective>

<tasks>

<task id="1" name="Add cache headers middleware helper">
<read_first>
- src/routes/route-utils.js
- src/routes/app.routes.js
</read_first>
<action>
In `src/routes/route-utils.js`, add a helper:

```javascript
/**
 * @param {number} maxAge - Cache duration in seconds
 * @returns {import('express').RequestHandler}
 */
function cacheControl(maxAge) {
  return (req, res, next) => {
    res.set('Cache-Control', `public, max-age=${maxAge}`);
    next();
  };
}
```

Export it alongside existing helpers.
</action>
<acceptance_criteria>
- `grep "cacheControl" src/routes/route-utils.js` matches
- `grep "Cache-Control" src/routes/route-utils.js` matches
</acceptance_criteria>
</task>

<task id="2" name="Apply cache headers to read-only GET routes">
<read_first>
- src/routes/app.routes.js
- src/routes/assistant-workspace.routes.js
- src/routes/assistant-git.routes.js
</read_first>
<action>
Apply `cacheControl()` middleware to appropriate GET routes:

1. `app.routes.js` — repo doc listing/rendering endpoints: `cacheControl(60)` (1 minute for dynamic content)
2. `assistant-workspace.routes.js` — file listing, file read endpoints: `cacheControl(30)` (30s, files change frequently)
3. `assistant-git.routes.js` — git status, log, diff endpoints: `cacheControl(10)` (10s, changes often)

Do NOT add to:
- SSE streaming endpoints (already have `Cache-Control: no-cache`)
- POST/PUT/PATCH/DELETE routes
- Auth routes (session-specific, never cacheable)

Usage pattern: `router.get('/path', requireAuth, cacheControl(60), handler)`
</action>
<acceptance_criteria>
- `grep -c "cacheControl" src/routes/app.routes.js` returns at least 1
- `grep -c "cacheControl" src/routes/assistant-workspace.routes.js` returns at least 1
- `npm test` passes
</acceptance_criteria>
</task>
