# Plan 22-03: E2E + Frontend Smoke Tests

<objective>
Create Playwright E2E test suite covering core user flows and frontend smoke tests for all 16 pages. Checkpoint: requires running server for E2E verification.
</objective>

<tasks>

<task id="1" name="Install Playwright and configure">
<read_first>
- package.json
</read_first>
<action>
Run `npm install --save-dev @playwright/test` and `npx playwright install chromium`.

Create `playwright.config.js`:
```javascript
const { defineConfig } = require('@playwright/test');
module.exports = defineConfig({
  testDir: './test/e2e',
  timeout: 30000,
  retries: 1,
  use: {
    baseURL: 'http://localhost:3000',
    headless: true,
  },
  webServer: {
    command: 'node --env-file .env.test src/server.js',
    port: 3000,
    reuseExistingServer: true,
    timeout: 30000,
  },
});
```

Add script: `"test:e2e": "npx playwright test"`
</action>
<acceptance_criteria>
- `grep "@playwright/test" package.json` matches
- File `playwright.config.js` exists
- `grep "test:e2e" package.json` matches
</acceptance_criteria>
</task>

<task id="2" name="Create E2E test files">
<read_first>
- src/routes/auth.routes.js
- src/routes/assistant-workspace.routes.js
</read_first>
<action>
Create E2E test files in `test/e2e/`:

1. `login.spec.js` — login flow: navigate to login, enter credentials, verify redirect to app
2. `workspace.spec.js` — workspace open: login, select workspace, verify file tree loads
3. `chat.spec.js` — chat send: login, open workspace, send chat message, verify response appears
4. `terminal.spec.js` — terminal launch: login, open workspace, launch terminal, verify xterm renders
5. `smoke.spec.js` — all 16 pages load: iterate page URLs, verify HTTP 200 (or redirect to login), check no JS console errors

Each test uses Playwright's `test()` and `expect()`. Auth tests may need a seed user or test credentials via env vars.
</action>
<acceptance_criteria>
- Files exist: `test/e2e/login.spec.js`, `test/e2e/workspace.spec.js`, `test/e2e/chat.spec.js`, `test/e2e/terminal.spec.js`, `test/e2e/smoke.spec.js`
- Each file imports from `@playwright/test`
- `smoke.spec.js` covers all 16 page URLs
</acceptance_criteria>
</task>

<task id="3" name="Verify E2E suite runs" autonomous="false">
<read_first>
- playwright.config.js
</read_first>
<action>
Human checkpoint: Run `npm run test:e2e` with a local server and verify:
1. Login flow completes
2. Workspace loads
3. All 16 pages return 200 or redirect (no 500s)
4. No unhandled JS errors in console

This requires a running server with test data, which may need manual setup.
</action>
<acceptance_criteria>
- `npm run test:e2e` completes
- Smoke tests pass for all 16 pages
- At least login and workspace E2E tests pass
</acceptance_criteria>
</task>

</tasks>

<verification>
