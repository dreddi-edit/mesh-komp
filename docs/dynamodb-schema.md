# DynamoDB Schema Reference

All table names are configured via environment variables with a `mesh-` prefix default.
See `src/config/index.js` for the full config key list and `secure-db.js` for access function implementations.

---

## Users Table

**Env var:** `MESH_DYNAMO_USERS_TABLE`  
**Default:** `${MESH_DYNAMO_TABLE_PREFIX}-users` (e.g. `mesh-users`)

### Key Schema

| Key | Attribute | Type | Notes |
|-----|-----------|------|-------|
| Partition key | `id` | String | UUID — generated by `crypto.randomUUID()` |

### Global Secondary Indexes

| Index name | Partition key | Sort key | Projection |
|------------|---------------|----------|------------|
| `email-index` | `email` (String) | — | ALL |

The email GSI is required because the primary access pattern during login is `getUserByEmail`, which queries by email rather than ID.

### Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | User UUID (also stored as `userId` for denormalized reads) |
| `userId` | String | Duplicate of `id` for legacy read paths |
| `email` | String | Normalized (lowercase, trimmed) email address |
| `name` | String | Display name |
| `role` | String | `"operator"` (default) |
| `passwordHash` | String | scrypt hash — `scrypt:N:r:p:salt:hash` format |
| `createdAt` | String | ISO 8601 timestamp |
| `updatedAt` | String | ISO 8601 timestamp |

### Access Patterns

| Pattern | Function | Key Used |
|---------|----------|----------|
| Get user by ID | `getUserById(userId)` | Table PK (`id`) |
| Get user by email | `getUserByEmail(email)` | GSI `email-index` |
| Create / update user | `upsertUser(user)` | Table PK (`id`) via PutCommand |

## Sessions Table

**Env var:** `MESH_DYNAMO_SESSIONS_TABLE`  
**Default:** `${MESH_DYNAMO_TABLE_PREFIX}-sessions` (e.g. `mesh-sessions`)

### Key Schema

| Key | Attribute | Type | Notes |
|-----|-----------|------|-------|
| Partition key | `id` | String | SHA-256 hash of the raw session token |

Raw tokens are never stored — only their hash. The raw token is kept only in the `httpOnly` cookie on the client.

### Global Secondary Indexes

| Index name | Partition key | Sort key | Projection |
|------------|---------------|----------|------------|
| `userId-index` | `userId` (String) | — | ALL |

Required for `listSessionsByUser` (session management UI) and `deleteSessionsByUser` (logout-all).

### Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | SHA-256 hash of the raw token |
| `userId` | String | Owner's user UUID |
| `createdAt` | Number | Unix timestamp (ms) |
| `lastSeenAt` | Number | Unix timestamp (ms) — updated on each authenticated request |
| `expiresAt` | Number | Unix timestamp (ms) — for app-level expiry check |
| `ttl` | Number | Unix timestamp (seconds) — DynamoDB TTL attribute for automatic deletion |
| `userAgent` | String | Browser/client User-Agent string |
| `ipAddress` | String | Client IP at session creation |
| `label` | String | Optional display label |

### TTL

The `ttl` attribute is configured as the DynamoDB TTL attribute. Expired sessions are automatically deleted by DynamoDB within 48 hours of expiry. The application also checks `expiresAt` on every request to reject sessions before DynamoDB has purged them.

### Access Patterns

| Pattern | Function | Key Used |
|---------|----------|----------|
| Read session by token | `readSession(rawToken)` | Table PK (`id` = hash of token) |
| Create session | `createSession(userId, ttlMs, metadata)` | Table PK (new hash) |
| Touch session (update lastSeenAt) | `touchSession(rawToken, ts)` | Table PK via UpdateCommand |
| Delete session by token | `deleteSession(rawToken)` | Table PK |
| List sessions by user | `listSessionsByUser(userId)` | GSI `userId-index` |
| Delete session by ID | `deleteSessionById(userId, sessionId)` | Table PK |
| Delete all sessions for user | `deleteSessionsByUser(userId, opts)` | GSI `userId-index` → batch deletes |

## User Store Table

**Env var:** `MESH_DYNAMO_STORES_TABLE`  
**Default:** `${MESH_DYNAMO_TABLE_PREFIX}-stores` (e.g. `mesh-stores`)

Used for per-user key-value storage: AI provider credentials (BYOK), settings, and preferences.

### Key Schema

| Key | Attribute | Type | Notes |
|-----|-----------|------|-------|
| Partition key | `id` | String | Composite key: `${userId}:${storeKey}` |

### Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | `${userId}:${storeKey}` composite |
| `userId` | String | Owner's user UUID |
| `storeKey` | String | Logical key (e.g. `"byok-credentials"`, `"user-settings"`) |
| `payloadEnc` | String | AES-256-GCM encrypted JSON payload (Base64-encoded) |
| `updatedAt` | String | ISO 8601 timestamp |

### Encryption

All `payloadEnc` values are encrypted with AES-256-GCM using a key derived from `MESH_DATA_ENCRYPTION_KEY`. The encrypted format is: `version(1 byte) + IV(12 bytes) + auth_tag(16 bytes) + ciphertext`. See `secure-db.js:encryptJson` / `decryptJson`.

### Access Patterns

| Pattern | Function | Key Used |
|---------|----------|----------|
| Set single value | `setUserStoreValue(userId, key, value)` | Table PK (composite) |
| Get single value | `getUserStoreValue(userId, key, fallback)` | Table PK (composite) |
| Get multiple values | `getUserStoreValues(userId, keys[])` | Table PK (composite × N) |

## Workspace Metadata Store (Cosmos DB / DynamoDB)

**Note:** The workspace metadata store (`workspace-metadata-store.cjs`) currently uses **Azure Cosmos DB** as the primary backend, not DynamoDB. It falls back to in-memory state when Cosmos is not configured.

**Cosmos DB container env vars:**

| Env var | Default | Purpose |
|---------|---------|---------|
| `MESH_COSMOS_ENDPOINT` | — | Cosmos DB endpoint URL |
| `MESH_COSMOS_KEY` | — | Cosmos DB access key |
| `MESH_COSMOS_DATABASE` | `mesh-db` | Database name |
| `MESH_COSMOS_WORKSPACE_FILES_CONTAINER` | `workspace_files` | Per-file metadata container |
| `MESH_COSMOS_WORKSPACES_CONTAINER` | `workspace_workspaces` | Per-workspace status container |

### Workspace Files Container

| Key | Attribute | Type | Notes |
|-----|-----------|------|-------|
| Document ID | `id` | String | `${workspaceId}:${normalizedPath}` |
| Partition key | `workspaceId` | String | Workspace identifier |

#### File Document Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | `workspaceId:path` composite |
| `workspaceId` | String | Workspace identifier |
| `path` | String | Normalized file path (no leading `/`, no `..`) |
| `status` | String | `pending \| processing \| completed \| failed \| ready \| partial` |
| `hash` | String | Content hash for change detection |
| `size` | Number | File size in bytes |
| `mimeType` | String | MIME type |
| `lastModified` | String | ISO 8601 timestamp |
| `capsuleBase` | String | Compressed capsule content (Brotli + base64) |
| `updatedAt` | String | ISO 8601 timestamp |

### Workspace Workspaces Container

| Key | Attribute | Type | Notes |
|-----|-----------|------|-------|
| Document ID / Partition key | `id` | String | Workspace identifier |

#### Workspace Document Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Workspace identifier |
| `status` | String | `processing \| completed \| failed \| ready \| partial` |
| `fileCount` | Number | Total file count |
| `completedCount` | Number | Files fully indexed |
| `updatedAt` | String | ISO 8601 timestamp |

### Access Patterns

| Pattern | Function | Key Used |
|---------|----------|----------|
| Get workspace status | `getWorkspace(workspaceId)` | Workspace container by ID |
| Set workspace status | `setWorkspace(workspaceId, data)` | Workspace container upsert |
| Get file metadata | `getFile(workspaceId, path)` | Files container by composite ID |
| Upsert file metadata | `setFile(workspaceId, path, data)` | Files container upsert |
| List workspace files | `listFiles(workspaceId, opts)` | Files container query by `workspaceId` partition |
| Batch upsert files | `setFiles(workspaceId, files[])` | Files container batch upsert |
| Delete workspace files | `deleteWorkspaceFiles(workspaceId)` | Files container scan + batch delete |
